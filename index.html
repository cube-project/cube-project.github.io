Software Architecture

No static architecture is given at deployement-time. Cube automatically creates and adapts the architecture depending on the actual known execution environment.

Easy Goals Specification

Cube provides a graphical, easy to use graph-based language to describe common administration objectives.

Decentralised Management

Cube platform consist of a set of autonomous Autonomic Managers where each one manages a part of the software system and collaborates with others to ensure dependent objectives.

Heterogeneity

Software systems are generally composed of various heterogeneious sub-systems. Cube allows the management of such systems using only one tool, with a predefined common goals.

Scalability

Cube can deal with a large number of managed resources, which can be highly distributed, and which can be exposed to a high frequency of changes.

Resilience

Cube provides self-healing and self-adaptation capabilities to ensure the survavibility of managed systems against failures and changine execution conditions.
