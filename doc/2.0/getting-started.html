Getting Started


Introduction

In this getting started tutorial, we will introduce you with the Cube platform with a simple, but a concret scenario. We will cover the main concepts and follow a step by step tasks to get the Cube platform self-managing a distributed component-based software system wile ensuring the resilience of the managed system.

Managed System

The managed system is examplified in the following picture:

The system is composed of a set of devices (Dev-*), a set of processing machines (PD-*) and a database. A special component "A" is deployed on the different devices to instantly collect particular type of data and send it to the processing machine. At this level, the data is filtred, aggregated and saved on a database.

Administration Objectives

To ensure the survavibility of the system, we have more that one processing machine, but only one in activity at a given time.

  1. At design-time, developers have not an idea about the running processing machines. Hence, designing a static architecture is not a possible. The component "A" should be connected to a another component "B" localted on any active processing machine (but only one at a time). The connection between "A" and "B" components should be done at runtime.
  2. When a processing machine fails, another PC of the cluster should replace it automatically. A new processing components should be created in this new machine, and all "A" components in the devices should be redirected to this new machine.
